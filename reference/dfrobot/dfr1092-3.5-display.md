## HEADLINE FINDING — Driver IC is ST7365P, NOT ILI9488

**The prior iteration's assumption that this display uses an ILI9488 driver is WRONG.** The datasheet PDF that DFRobot itself links from the DFR1092 wiki page (`DFR1092_480-320-ips-screen_datasheet_1.0.pdf`) is, verbatim, the **Sitronix ST7365P** datasheet:

> "ST7365P — 320RGB x 480 dot 16.7M Color with Frame Memory — Single-Chip TFT Controller/Driver — Datasheet ... © 2013 Sitronix Technology Corporation."

This was confirmed by downloading the PDF directly and extracting its text (not just trusting a page summary) — the title page, part number, and section headers throughout the 14,700+ line document all say ST7365P/Sitronix. The panel resolution quoted in the datasheet (320 x 480) matches the DFR1092's native panel resolution exactly (DFRobot's own product description says *"It features a 320×480 IPS panel for 178° distortion-free full viewing"* — the "480×320" in the SKU/marketing name is the landscape-rotated figure).

Corroborating evidence:
- DFRobot's own `DFRobot_GDL` library (the library named as compatible with this display on the wiki) ships a driver class literally named `DFRobot_ST7365P_320x480_HW_SPI` (and a `..._DMA_SPI` DMA variant) — an exact resolution match for this panel — alongside separate classes for ILI9341/ILI9488/ST7789/ST7735S/ST7789/SSD1306 that do NOT match this panel's native resolution.
- **Likely source of the original ILI9488 assumption**: DFRobot sells a different, older, visually similar product — **DFR0669** ("Fermion: 3.5" 480x320 TFT LCD Capacitive Touchscreen with MicroSD Card Slot (Breakout)") — which genuinely IS ILI9488-driven, has the same 3.5"/480x320 spec, but uses a plain pin-header breakout (no GDI connector) and is a completely separate SKU. DFR0669 appears as a "related product" recommendation embedded in the DFR1092 wiki page's own page data, which is almost certainly what caused the mix-up. **Do not use ILI9488 init code, MADCTL values, or gamma tables for the DFR1092 — use ST7365P.**

## Display Driver IC — CONFIRMED

**ST7365P** (Sitronix), a single-chip TFT controller/driver with on-chip frame memory, supporting 16.7M colors, native 320RGB x 480 dot panel. Supports both 3-line and 4-line SPI serial interface modes (datasheet sections 8.4.1 "3-Line Interface" and 8.4.2 "4-Line Interface"). The DFR1092 board is wired for **4-line SPI** (separate D/C line broken out, per the pinout below).

## Interface Type

- **SPI** (4-wire: SCLK, MOSI, MISO, CS + separate D/C and RST lines — i.e. the common "4-line" TFT SPI convention with 6 total control signals: SCK/MOSI/MISO/CS/DC/RST)
- Not parallel/RGB — this is a serial-only interface.
- Two physical connector options on the board:
  1. **GDI (Gravity Display Interface)** — DFRobot-proprietary 18-pin FPC connector, solder-free, single-cable, designed to plug directly into FireBeetle 2 boards (including the ESP32-C6 V1.2 in this project).
  2. **Standard 2.54 mm pin header** — 14 pins, for use with any generic MCU/Arduino via jumper wires.

## Touch Controller

- **Type**: Capacitive
- **Touch points**: 5-point multi-touch
- **Interface**: **I2C** — confirmed directly from the schematic (net labels `CTP_SCL`, `CTP_SDA`, `CTP_INT`, `CTP_RST` on the touch-panel FPC connector; also exposed as `SCL`/`SDA`/`INT`/`TP_R`(reset) on the 2.54 mm header and `SCL`/`SDA`/`INT`/`TCS` on the GDI connector).
- **Exact touch IC part number: NOT confirmed by a primary DFRobot source.** The wiki page, product page, datasheet, and the single-page schematic available from DFRobot all stop short of naming it (the schematic only shows a connector out to a separate touch-panel FPC/module; the touch chip itself lives on that module and isn't itemized in the documents DFRobot publishes for this SKU).
- Since the interface is I2C (not SPI), the touch IC is **not** the XPT2046 (SPI, resistive, single-touch) that `DFRobot_GDL` also supports. Of the touch drivers the library ships (`XPT2046`, `GT911`, `GT5688`), the I2C capacitive candidates are **GT911** or **GT5688** (both Goodix-family, both I2C, both multi-touch capable).
- A secondary (non-DFRobot-sourced, AI-search-synthesized) result claimed the DFR1092 specifically uses **GT911 at I2C address 0x5D**. This could not be traced to an actual DFRobot document, datasheet, or schematic text and should be treated as an **unverified lead, not a confirmed fact** — it may simply reflect GT911 being the generic, most common chip of its class rather than a DFR1092-specific citation. GT911 is the leading candidate on general-plausibility grounds (I2C, 5-point capacitive, extremely common in this display-size class, and already supported by DFRobot's own GDL library), but treat it as unconfirmed until verified against the physical unit's chip marking or direct DFRobot support confirmation.

## Software / Library

- **Library name**: `DFRobot_GDL` (DFRobot's own graphics/display/touch/UI framework)
- GitHub: https://github.com/DFRobot/DFRobot_GDL
- Requires Arduino IDE 1.8.10 or later.
- Relevant driver class for this panel: **`DFRobot_ST7365P_320x480_HW_SPI`** (a DMA-SPI variant also exists for supported MCUs). Do **not** use `DFRobot_ILI9488_320x480_HW_SPI` — that class is for the unrelated DFR0669/other ILI9488 panels, not this one.
- Touch driver classes shipped in the same library: `XPT2046`, `GT911`, `GT5688` — exact class to use for DFR1092 not yet confirmed (see Touch Controller section above).
- Example sketches referenced on the wiki's "Docs" tab:
  - **basicTest** — basic drawing primitives (points, lines, shapes)
  - **gestureFont** (labeled "gettureFont" on the page, likely a typo) — UI numeric-keypad demo
  - **UI gesture** — gesture-recognition demo
- Wiki guidance quoted verbatim: *"When connecting the screen with an FPC cable, simply configure the corresponding pin numbers according to the GDL demo. Normally, only three pins need to be configured on different main controllers"* — implying most of the example sketches only require CS/DC/RST to be set per-board, with SCLK/MOSI/MISO/power fixed by the GDI connector wiring.

## Power Requirements

| Parameter | Value |
|---|---|
| Operating voltage | DC 3.3 V ~ 5 V |
| Operating current | 50 mA @ 5 V |
| Backlight control | Digital level on `LCD_BL` (GDI) / `BLK` (2.54 mm header) pin. Has a default on-state — the screen lights up even if the backlight pin is left unconnected. Per the wiki: **high level (1) = maximum brightness, low level (0) = backlight off.** The wiki only documents this as a digital high/low control; it does not state whether PWM dimming is supported. However, the schematic shows the `BLK` net driving a transistor (with base/current-limiting resistors), i.e. a transistor-switched LED backlight — this topology commonly tolerates a PWM signal on the control pin for dimming, but that is an inference from the schematic, not a documented, DFRobot-confirmed feature. Treat as digital on/off unless PWM dimming is tested and confirmed on real hardware. |

Onboard regulation (from schematic): an **ME6206A30XG** LDO regulates the incoming 3.3–5 V rail down to a local 3 V rail for the panel logic; an **TXS0108E** 8-bit bidirectional level shifter is present on the SPI/control lines (for logic-level translation between host and panel domains).

## Open Items / Flags for Firmware Bring-Up

1. **Driver IC corrected to ST7365P** — any ported firmware, `LovyanGFX`/`TFT_eSPI` config, init sequence, or MADCTL/gamma table written under the ILI9488 assumption must be replaced with ST7365P-specific configuration (or use DFRobot's own `DFRobot_GDL` / `DFRobot_ST7365P_320x480_HW_SPI` class directly, which is the officially supported path).
2. **Exact touch IC model unconfirmed** — leading candidate is GT911 (I2C, capacitive, 5-point), with GT5688 as the other library-supported alternative; an unverified secondary source claims GT911 @ 0x5D but this isn't traceable to a DFRobot primary source. Recommend checking the physical touch-panel FPC/chip marking on the real unit once it arrives, or asking DFRobot support, before finalizing touch-driver code.
3. Watch for the DFR0669 (ILI9488) vs DFR1092 (ST7365P) mix-up in any future research — they are easy to conflate because DFRobot's own site cross-links them as "related products" and both are nominally "3.5-inch 480x320 capacitive touch" panels.
