Wiring — Display to Firebeetle C6

✓  These GPIO assignments match the Firebeetle C6 GDI connector exactly. When the DFR1092 display arrives, swap the display module — no rewiring needed on the Firebeetle side.
Display Pin Firebeetle C6 Pin GPIO Type Notes
VCC → 3V3 — POWER 3.3 V supply only — do not use 5 V
GND → GND — POWER Any GND pin
DIN → MOSI GPIO22 SPI SPI data out (host → display)
CLK → SCK GPIO23 SPI SPI clock
CS → D6 GPIO1 CTRL Display chip select (active LOW)
DC → D2 GPIO8 CTRL Data / Command select
RST → D3 GPIO14 CTRL Hardware reset (active LOW)
BL → D13 GPIO15 CTRL Backlight enable. Also onboard LED — backlight will mirror LED state
⚠  No MISO connection needed. The ILI9341 is write-only in this configuration. Leave the Firebeetle MISO (GPIO21) unconnected. Also no touch on this module — leave TOUCH_CS (GPIO6) unconnected.

Gicar Serial Port Wiring (CN11 → Firebeetle C6)

ℹ  The Gicar UART was on GPIO16/17 on the original ESP32 board. Those pins are not broken out on the Firebeetle C6 header, so the UART is remapped to GPIO5 (TX) and GPIO4 (RX). The button moves from GPIO14 (now LCD_RST) to GPIO10.
CN11 Signal Firebeetle C6 Pin GPIO Notes
+12 V → Buck converter in — 12 V → 5 V buck → Firebeetle 5V pin (same as original board)
GND → GND — Common ground
TXD (machine sends) → D4 GPIO4 GICAR_RX_PIN — machine TXD → ESP32 RX. Add 10 kΩ pull-up to 3.3 V (2.5 V signal is marginal for VIH)
RXD (machine receives) → D5 GPIO5 GICAR_TX_PIN — ESP32 TX → machine RXD
⚠  10 kΩ pull-up required on RX (GPIO4). CN11 TXD idles at 2.5 V LVCMOS — below ESP32-C6's guaranteed VIH. A 10 kΩ resistor from GPIO4 to 3.3 V ensures reliable logic-high detection. The TX line (GPIO5) needs no pull-up.
Button — moved to GPIO10
SignalFirebeetle C6 PinGPIONotes
Button 1 (active LOW) → D10 GPIO10 Moved from GPIO14 (now LCD_RST). Wire between GPIO10 and GND — INPUT_PULLUP in firmware
Button 2 (active LOW) → D0 GPIO2 Wire between GPIO2 and GND — INPUT_PULLUP in firmware

platformio.ini — Changes Required for This Display

ℹ  These settings are for the temporary 2.4" ILI9341. When the DFR1092 3.5" display arrives, change back to ILI9488_DRIVER, TFT_WIDTH=320, TFT_HEIGHT=480. 
; platformio.ini — temporary ILI9341 build for Firebeetle C6 bring-up

[env:lm_controller]
platform  = espressif32
board     = esp32-c6-devkitc-1    ; Firebeetle C6 — closest matching board
framework = arduino

monitor_speed = 115200

board_build.partitions = huge_app.csv
board_build.filesystem = littlefs

build_flags =
    -DCORE_DEBUG_LEVEL=0
    -I include
    -DLV_CONF_INCLUDE_SIMPLE
    ; ── TFT_eSPI: ILI9341 2.4" SPI display (240×320 portrait native) ──
    -DUSER_SETUP_LOADED=1
    -DILI9341_DRIVER=1
    -DTFT_WIDTH=240
    -DTFT_HEIGHT=320
    -DTFT_MOSI=22     ; MOSI — Firebeetle C6 GDI / DIN on display
    -DTFT_SCLK=23     ; SCK  — Firebeetle C6 GDI / CLK on display
    -DTFT_CS=1        ; D6   — Firebeetle C6 GDI / CS on display
    -DTFT_DC=8        ; D2   — Firebeetle C6 GDI / DC on display
    -DTFT_RST=14      ; D3   — Firebeetle C6 GDI / RST on display
    -DTFT_BL=15       ; D13  — Firebeetle C6 GDI / BL on display
    -DTFT_BACKLIGHT_ON=HIGH
    ; No TOUCH_CS — this display has no touch
    -DLOAD_GLCD=1
    -DLOAD_FONT2=1
    -DLOAD_FONT4=1
    -DLOAD_FONT6=1
    -DLOAD_FONT7=1
    -DLOAD_FONT8=1
    -DLOAD_GFXFF=1
    -DSMOOTH_FONT=1

When DFR1092 3.5" GDI Display Arrives

✓  Wiring on the Firebeetle side is identical — no rewiring needed. Plug the 18-pin FPC into the GDI connector on the Firebeetle C6.
platformio.ini changes for DFR1092:
  • Change ILI9341_DRIVER → ILI9488_DRIVER
  • Change TFT_WIDTH=240 → TFT_WIDTH=320
  • Change TFT_HEIGHT=320 → TFT_HEIGHT=480
  • Add -DTOUCH_CS=6 (GT911 uses I2C not SPI — see touch.cpp note below)
  • Restore SCREEN_W=480, SCREEN_H=320 in main.cpp

touch.cpp changes for DFR1092:
  • Replace XPT2046 (SPI resistive) with GT911 (I2C capacitive) driver
  • GT911 address: 0x5D or 0x14 — confirm with I2C scan
  • I2C pins: SDA=GPIO19, SCL=GPIO20 (already on Firebeetle header)
